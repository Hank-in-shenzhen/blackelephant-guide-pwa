## 1. 项目概述

### 1.1 背景

黑象小酒馆是位于深圳罗湖的一家主理人驱动型社交酒吧，与泰餐厅共享空间运营。随着业务发展，需要一套高效的新员工培训工具，降低主理人的教学成本，同时保证出品质量的一致性。此外，小吃类产品的推出和库存管理的需求也日益迫切。

### 1.2 目标

- 新员工能够独立查询任意鸡尾酒、小吃的完整 SOP，无需反复询问主理人
- 员工通过学习/实操两种模式，兼顾学习深度与操作效率
- 每日开工、打烊流程标准化，通过 Checklist 确保不遗漏任何步骤
- 主理人可随时在后台更新 SOP 和 Checklist，员工端实时同步
- 建立完整的库存管理体系，支持库存盘点、预警和补货提醒

### 1.3 核心价值

> 「把老板从重复性教学中解放出来，把员工从记忆负担中解放出来，把库存管理从混乱中解放出来」
>
> 核心模型：主理人维护内容，员工随时查阅，系统保证一致性，库存数字化管理。

## 2. 用户与使用场景

### 2.1 用户角色

| 角色 | 说明 |
|------|------|
| 员工（默认端） | 新入职员工，主要在吧台使用，手机操作为主 |
| 主理人（管理端） | 不老(Hank)，负责维护 SOP 和 Checklist 内容 |
| 库管员 | 负责库存盘点、管理和预警处理（可由主理人兼任） |

### 2.2 使用场景

**场景 A · 上班前学习**
员工坐下来，打开 SOP 查询页，进入「学习模式」，完整阅读一款酒或小吃的配方、步骤、注意事项，并点击视频链接跳转小红书观看教程。

**场景 B · 实操备忘**
员工正在调酒或制作小吃，需要快速确认用量或步骤顺序。切换至「实操模式」，信息精简、字体偏大，站着扫一眼即可，不打断操作节奏。

**场景 C · 顾客推荐**
顾客说「我不喜欢太烈的，有没有果味的？」员工用风味/酒精感筛选，快速找到合适推荐，并能用一两句话介绍口感（来自风味标签）。顾客想点小吃时，也能快速查看小吃SOP进行推荐。

**场景 D · 开工/打烊**
员工到店后打开开工清单，逐项勾选，进度条实时更新。全部完成显示确认提示。打烊同理。每天自动重置，不需要手动操作。

**场景 E · 主理人更新内容**
不老新增一款招牌酒或小吃，打开 /admin 输入密码，填写配方/步骤/视频链接，保存后员工端下次刷新即可看到。

**场景 F · 库存盘点**
库管员打开库存管理功能，查看库存预警信息，发起盘点任务，逐项核对库存数量，录入盘点结果，生成盘点报告。补货时更新库存记录。

**场景 G · 小吃制作**
顾客点了一份小吃，员工打开小吃SOP，查看制作步骤和注意事项，按SOP完成出品。

## 3. 功能需求

### 3.1 员工端 · 鸡尾酒 SOP 查询

| 功能点 | 说明 |
|--------|------|
| 实时搜索 | 输入框防抖 300ms，匹配酒名、食材、风味标签，中英文均可 |
| 分类筛选 | 横向滚动 Chips：全部 / 长饮 / 短饮 / 招牌 |
| 酒精感筛选 | 轻 / 中 / 烈 三档，色块标识（绿/金/红） |
| 风味筛选 | 清爽 / 果香 / 花香 / 茶香 / 微苦 / 酸甜 / 烟熏 等 |
| 卡片列表 | 展示酒名、英文名、酒精感标签、风味标签 |
| SOP 详情页 | 点击卡片展开，展示完整信息 |
| 学习 / 实操切换 | 每张卡片独立切换，状态本地保存 |
| 视频跳转 | 点击按钮新标签页打开小红书链接 |
| 无结果提示 | 搜索无结果时显示友好提示 |

#### 3.1.1 学习模式

- 完整原料清单：名称 + 用量 + 备注
- 完整步骤：带序号，包含所有解释说明
- 注意事项（如有）
- 视频链接按钮

#### 3.1.2 实操模式

- 原料以卡片宫格形式展示：上方原料名，下方用量，字体偏大
- 步骤去除解释文字，只保留动作指令，序号改为金色圆形标
- 关键数字（用量、时间）保持完整，不做任何删减
- 注意事项保留
- 视频链接按钮保留

## 4. 数据结构

### 4.1 鸡尾酒 SOP 条目
> 存储于 Supabase · `cocktail_sop` 表

```
id            UUID (主键)
name          string  酒名（中文）
name_en       string  英文名（可选）
category      string  长饮/短饮/招牌
strength      string  轻/中/烈
flavor_tags   array   风味标签数组
ingredients   array   [{name, amount, note}]
steps         array   步骤字符串数组
notes         string  注意事项（可选）
video_url     string  小红书/Bilibili 链接
created_at    timestamp
updated_at    timestamp
```

### 4.2 小吃 SOP 条目
> 存储于 Supabase · `snack_sop` 表

```
id            UUID (主键)
name          string  小吃名（中文）
category      string  炸物/小食/主食/招牌小吃
taste_tags    array   口味标签数组
make_time     string  快速/中等/慢
ingredients   array   [{name, amount, note}]
equipment     array   所需设备列表
steps         array   步骤字符串数组
notes         string  注意事项（可选）
storage_tip   string  保存温度/保质期提示（可选）
video_url     string  小红书/Bilibili 链接
created_at    timestamp
updated_at    timestamp
```

### 4.3 Checklist 模板
> 存储于 Supabase · `checklist_templates` 表

```
id       UUID
type     string  open / close
title    string  任务描述
order    int     排序权重
```

### 4.4 每日打卡记录
> 存储于 Supabase · `checklist_logs` 表

```
id       UUID
date     date    打卡日期
type     string  open / close
items    jsonb   [{template_id, title, done}]
```

### 4.5 库存物品
> 存储于 Supabase · `inventory_items` 表

```
id              UUID (主键)
name            string  物品名称
category        string  酒类/配料/小吃原料/耗材/其他
unit            string  单位（瓶/毫升/克/个/包等）
current_stock   number  当前库存数量
safety_stock    number  安全库存（低于此值预警）
last_updated    timestamp
notes           string  备注（可选）
barcode         string  条码/二维码（可选）
```

### 4.6 库存变动记录
> 存储于 Supabase · `inventory_logs` 表

```
id              UUID (主键)
item_id         UUID    关联 inventory_items.id
change_type     string  in/out/adjust  入库/出库/调整
quantity        number  变动数量（正数为增加，负数为减少）
balance         number  变动后结余
reason          string  变动原因
operator        string  操作人
created_at      timestamp
```

### 4.7 盘点任务
> 存储于 Supabase · `inventory_checks` 表

```
id              UUID (主键)
check_date      date    盘点日期
status          string  pending/done  进行中/已完成
operator        string  盘点人
scope           string  all/category  全部/分类
category        string  盘点分类（scope为category时必填）
created_at      timestamp
completed_at    timestamp  完成时间（status为done时）
```

### 4.8 盘点明细
> 存储于 Supabase · `inventory_check_items` 表

```
id              UUID (主键)
check_id        UUID    关联 inventory_checks.id
item_id         UUID    关联 inventory_items.id
system_stock    number  系统库存数量
actual_stock    number  实际盘点数量
difference      number  差异数量
notes           string  备注（可选）
```

## 5. 技术方案

### 5.1 技术栈

| 层级 | 选型 |
|------|------|
| 前端框架 | React + Vite |
| 样式 | TailwindCSS |
| PWA | vite-plugin-pwa（支持添加到手机主屏幕） |
| 路由 | react-router-dom |
| 云端数据库 | Supabase 免费层（500MB PostgreSQL） |
| 本地缓存 | localStorage（离线降级） |
| 部署 | Vercel / GitHub Pages（免费） |
| 开发工具 | Claude Code CLI |

### 5.2 存储策略

- 主数据存储于 Supabase，所有设备实时同步
- 本地 localStorage 作为缓存，网络不可用时降级使用
- `storage.js` 统一封装读写逻辑，组件层不感知存储方式
- Supabase 凭证通过环境变量注入，不写入代码

### 5.3 PWA 配置

- manifest.json 配置酒杯图标，主题色 #C9A84C
- 支持「添加到主屏幕」，体验接近原生 App
- Service Worker 缓存静态资源，首屏离线可用

### 7.3 开工清单（9项）

1. 摆桌灯、点餐牌，介绍消费100送50鸡尾酒券活动（重要！）
2. 打开电视墙氛围灯，亮度调最高（用遥控器）
3. 更换门口宣传KT板
4. 检查柠檬汁是否解冻（提前一天检查最重要！）
5. 检查其他调酒原材料库存，备料
6. 摆放烟灰缸、纸巾、骰盅
7. 扫地面，摆桌椅
8. 展示柜补啤酒
9. 检查快递：冻品及时放进冰箱，其他原料按标准摆放

### 7.4 打烊清单（8项）

1. 清洗 shake 壶、量杯等调酒器具
2. 清洗空气炸锅
3. 洗杯子消毒
4. 精酿沥水垫归位
5. 明日柠檬汁放入保鲜冰箱解冻
6. 再次检查库存，记录缺货
7. 展示柜补满啤酒
8. 关空调、关电视、关灯、关门

## 10. 待定与开放问题

- 盘点报告是否需要支持打印或导出为PDF

## 1. 项目概述

### 1.1 背景

黑象小酒馆是位于深圳罗湖的一家主理人驱动型社交酒吧，与泰餐厅共享空间运营。随着业务发展，需要一套高效的新员工培训工具，降低主理人的教学成本，同时保证出品质量的一致性。

### 1.2 目标

- 新员工能够独立查询任意鸡尾酒的完整 SOP，无需反复询问主理人
- 员工通过学习/实操两种模式，兼顾学习深度与操作效率
- 每日开工、打烊流程标准化，通过 Checklist 确保不遗漏任何步骤
- 主理人可随时在后台更新 SOP 和 Checklist，员工端实时同步

### 1.3 核心价值

> 「把老板从重复性教学中解放出来，把员工从记忆负担中解放出来」
>
> 核心模型：主理人维护内容，员工随时查阅，系统保证一致性。

## 2. 用户与使用场景

### 2.1 用户角色

| 角色 | 说明 |
|------|------|
| 员工（默认端） | 新入职员工，主要在吧台使用，手机操作为主 |
| 主理人（管理端） | 不老(Hank)，负责维护 SOP 和 Checklist 内容 |

### 2.2 使用场景

**场景 A · 上班前学习**
员工坐下来，打开 SOP 查询页，进入「学习模式」，完整阅读一款酒的配方、步骤、注意事项，并点击视频链接跳转小红书观看教程。

**场景 B · 实操备忘**
员工正在调酒，需要快速确认用量或步骤顺序。切换至「实操模式」，信息精简、字体偏大，站着扫一眼即可，不打断操作节奏。

**场景 C · 顾客推荐**
顾客说「我不喜欢太烈的，有没有果味的？」员工用风味/酒精感筛选，快速找到合适推荐，并能用一两句话介绍口感（来自风味标签）。

**场景 D · 开工/打烊**
员工到店后打开开工清单，逐项勾选，进度条实时更新。全部完成显示确认提示。打烊同理。每天自动重置，不需要手动操作。

**场景 E · 主理人更新内容**
不老新增一款招牌酒，打开 /admin 输入密码，填写配方/步骤/视频链接，保存后员工端下次刷新即可看到。

## 3. 功能需求

### 3.1 员工端 · SOP 查询

| 功能点 | 说明 |
|--------|------|
| 实时搜索 | 输入框防抖 300ms，匹配酒名、食材、风味标签，中英文均可 |
| 分类筛选 | 横向滚动 Chips：全部 / 长饮 / 短饮 / 招牌 |
| 酒精感筛选 | 轻 / 中 / 烈 三档，色块标识（绿/金/红） |
| 风味筛选 | 清爽 / 果香 / 花香 / 茶香 / 微苦 / 酸甜 / 烟熏 等 |
| 卡片列表 | 展示酒名、英文名、酒精感标签、风味标签 |
| SOP 详情页 | 点击卡片展开，展示完整信息 |
| 学习 / 实操切换 | 每张卡片独立切换，状态本地保存 |
| 视频跳转 | 点击按钮新标签页打开小红书链接 |
| 无结果提示 | 搜索无结果时显示友好提示 |

#### 3.1.1 学习模式

- 完整原料清单：名称 + 用量 + 备注
- 完整步骤：带序号，包含所有解释说明
- 注意事项（如有）
- 视频链接按钮

#### 3.1.2 实操模式

- 原料以卡片宫格形式展示：上方原料名，下方用量，字体偏大
- 步骤去除解释文字，只保留动作指令，序号改为金色圆形标
- 关键数字（用量、时间）保持完整，不做任何删减
- 注意事项保留
- 视频链接按钮保留

### 3.2 员工端 · 开工/打烊清单

| 功能点 | 说明 |
|--------|------|
| 两个独立清单 | 开工清单 / 打烊清单，底部 Tab 独立切换 |
| Checkbox 交互 | 点击勾选/取消，状态实时保存 |
| 进度条 | 顶部显示 X/X 项完成，实时更新 |
| 完成提示 | 全部勾选后显示完成动画提示 |
| 每日自动重置 | 基于日期判断，新的一天自动从模板重新生成 |
| 今日日期显示 | 页面顶部显示当前日期 |

### 3.3 管理端 · /admin

| 功能点 | 说明 |
|--------|------|
| 密码保护 | 访问 /admin 路由弹出密码输入框，通过后进入 |
| SOP 列表管理 | 展示所有 SOP，支持新增、编辑、删除 |
| SOP 表单字段 | 酒名、英文名、分类、风味标签、酒精感、原料、步骤、注意事项、视频链接 |
| Checklist 管理 | 分开工/打烊两个列表，支持新增、删除、排序 |
| 实时生效 | 保存后员工端刷新即可看到最新内容 |

## 4. 数据结构

### 4.1 SOP 条目
> 存储于 Supabase · `sop_list` 表

```
id            UUID (主键)
name          string  酒名（中文）
name_en       string  英文名（可选）
category      string  长饮/短饮/招牌
strength      string  轻/中/烈
flavor_tags   array   风味标签数组
ingredients   array   [{name, amount}]
steps         array   步骤字符串数组
notes         string  注意事项（可选）
video_url     string  小红书/Bilibili 链接
created_at    timestamp
```

### 4.2 Checklist 模板
> 存储于 Supabase · `checklist_templates` 表

```
id       UUID
type     string  open / close
title    string  任务描述
order    int     排序权重
```

### 4.3 每日打卡记录
> 存储于 Supabase · `checklist_logs` 表

```
id       UUID
date     date    打卡日期
type     string  open / close
items    jsonb   [{template_id, title, done}]
```

## 5. 技术方案

### 5.1 技术栈

| 层级 | 选型 |
|------|------|
| 前端框架 | React + Vite |
| 样式 | TailwindCSS |
| PWA | vite-plugin-pwa（支持添加到手机主屏幕） |
| 路由 | react-router-dom |
| 云端数据库 | Supabase 免费层（500MB PostgreSQL） |
| 本地缓存 | localStorage（离线降级） |
| 部署 | Vercel / GitHub Pages（免费） |
| 开发工具 | Claude Code CLI |

### 5.2 存储策略

- 主数据存储于 Supabase，所有设备实时同步
- 本地 localStorage 作为缓存，网络不可用时降级使用
- `storage.js` 统一封装读写逻辑，组件层不感知存储方式
- Supabase 凭证通过环境变量注入，不写入代码

### 5.3 PWA 配置

- manifest.json 配置酒杯图标，主题色 #C9A84C
- 支持「添加到主屏幕」，体验接近原生 App
- Service Worker 缓存静态资源，首屏离线可用

### 5.4 目录结构

```
src/
  components/
    BottomNav.jsx
    SOPList.jsx
    SOPDetail.jsx
    Checklist.jsx
    Admin/
      AdminAuth.jsx
      SOPManager.jsx
      ChecklistManager.jsx
  hooks/
    useLocalStorage.js
    useSearch.js
    useSupabase.js
  data/
    seedData.js       # 22款酒 + 开工/打烊清单初始数据
  utils/
    storage.js        # 统一存储封装（Supabase + localStorage降级）
    date.js
    supabase.js
  pages/
    SOPPage.jsx
    ChecklistPage.jsx
    AdminPage.jsx
  App.jsx
  main.jsx
```

### 7.2 开工清单（9项）

1. 摆桌灯、点餐牌，介绍消费100送50鸡尾酒券活动（重要！）
2. 打开电视墙氛围灯，亮度调最高（用遥控器）
3. 更换门口宣传KT板
4. 检查柠檬汁是否解冻（提前一天检查最重要！）
5. 检查其他调酒原材料库存，备料
6. 摆放烟灰缸、纸巾、骰盅
7. 扫地面，摆桌椅
8. 展示柜补啤酒
9. 检查快递：冻品及时放进冰箱，其他原料按标准摆放

### 7.3 打烊清单（8项）

1. 清洗 shake 壶、量杯等调酒器具
2. 清洗空气炸锅
3. 洗杯子消毒
4. 精酿沥水垫归位
5. 明日柠檬汁放入保鲜冰箱解冻
6. 再次检查库存，记录缺货
7. 展示柜补满啤酒
8. 关空调、关电视、关灯、关门
